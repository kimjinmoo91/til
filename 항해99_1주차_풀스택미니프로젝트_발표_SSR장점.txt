SSR의 장점을 제출하는 과제가 있었다. REACT SSR만 검색을 해보다가 제출을 위해 각자 공부하면서 과제를 준비한 것에 이야기를 나눠보는 시간이 있었는데,
단편적인, 초등 수준의 장점만을 생각하고, 공부하고, 검색을 해보았었다는 것을 팀원의 공부를 통해 알게 되었다.

기초가 없는 터라 받은 공부 내용을 전부 이해하기는 어려웠지만 어떤 흐름인지는 알고 있다.

-SSR 장점-

- SERVER에서 모든 HTML에 필요한 데이터들을 완성시킨 후, CLIENT에 제공을 한다.
  CSR과 비교하여 다운 받아야하는 JS 파일의 수가 적기 때문에 '초기 로딩 속도'가 느리다.
-모든 데이터를 갖는 HTML 파일을 CLIENT로 제공하기 때문에 검색엔진에 노출될 확률 증가.
  CSR과 비교할 때 CSR은 데이터를 갖고 오기 위해 사용자와 상호작용 하기 전에 데이터가 존재하지 않으므로 검색엔진에 노출 될 확률이 감소.

=> 1. 인증을 위한 별도의 저장소가 필요 X ->분산 시스템에서의 확장성이 우수.(서버가 분산되어 있을 경우에도 인증 가능.)
     2. 별도의 I/O 작업 없이 TOKEN을 DECODE해서 SIGNATURE를 통해 위변조 파악 가능.
     3. TOKEN 검증을 위한 정보가 TOKEN 내부 존재.
